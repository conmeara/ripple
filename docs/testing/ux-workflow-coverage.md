# Ripple V1 UX Workflow Coverage Matrix

This matrix maps expected user journeys to regression evidence. A workflow is
allowed to be `Release-gated` only when it requires a packaged app, external
provider credentials, PostHog official-build configuration, signing/notarization
secrets, or real render/browser resources that should not run on every commit.

Status meanings:

- `Automated`: covered by local deterministic tests.
- `Automated + release smoke`: local tests cover behavior and the release
  checklist keeps a packaged/manual smoke for artifact-specific risk.
- `Release-gated`: must be proven by `docs/release/v1-release-checklist.md`.
