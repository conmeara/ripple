Ripple · v0.13.0 · what it is and how it works

One perception cache, eighteen instruments,
four opinions on the craft.

Everything in Ripple hangs off one spine: perceive → decide-with-user → assemble → verify → deliver. The CLI is the product and teaches itself through stdout; the skills are the optional craft opinion on top; taste lives in project files, not prompts. Click any command or skill below to open it.

Structure

The spine, and the three layers that serve it

Perceive

Build eyes and ears once; cache forever. Every later question is a cheap query.

analyzetranscribeprobeframe-sheettimeline-sheet

Decide

Find the cut with the user: search takes, verify endpoints, judge with three signals.

searchselectcandidatessyncstudy

Assemble

Decisions become a machine-checkable paper edit, then a render.

edit.jsonlintcutbeats

Verify

Deterministic gates plus an independent read-only reviewer. Editors don't self-grade.

qaqa-reviewerhistory

Deliver

A rendered file — or hand the structure to an NLE as a peer terminal.

captionshandoff

All four scenarios (raw-clips → cut, demo video, interview + b-roll, fiction) run this same spine — the per-scenario differences are almost entirely opinions, which is why the playbooks are thin and the CLI is shared.

Tools

What each command does, and how it works

The unit of value is the (instrument, judgment) pair: a measurement plus the editorial decision it enables. Each entry says what the instrument measures and how — expand any command.

Core loop

raw clips → verified render · 7 commands
analyzebuild the perception index — run once per source

What it doesThe eyes and ears. One pass produces the cached index: word timings, measured silence at multiple thresholds, sentences with pace and terminal pitch, fillers, breaths, audible non-speech events (laughs, claps), scene changes, motion and energy curves, and a drift self-check.

How it worksExtracts 16kHz mono WAV; whisper-cpp transcribes it — long sources are pre-segmented at measured silences into ≤30s chunks with timestamps offset-added, so cumulative drift is structurally impossible (v0.11). Word timing comes from a second whisper pass (-ml 1 -sow), fused with ffmpeg silencedetect; prosody, motion, and scene curves come from ffmpeg filters. Cached by file content hash, so moved files keep their index.

Reads → writesReads the media file → writes work/analysis/<hash>.analysis.json (+ 16k WAV, word JSONs). Everything downstream queries this cache instead of re-perceiving.

candidatesverify a cut range with three signals before you trust it

What it doesThe endpoint instrument for "cut after the completed thought." For a proposed IN/OUT it returns word timing (lastWordEnd, tailGap, what starts next), measured cut-safety flags, a mechanical suggestedOut and suggestedIn, silence at multiple thresholds, the range transcript, edge frames, and head/tail cut-card sheets.

How it worksReads the cached index; computes timing via cutTiming; flags DEAD_AIR_TAIL, LATE_FIRST_WORD, STUTTER_CUT (cut inside a <0.25s silence), MICRO_CLIP (<1s kept range) and more; suggestedOut = lastWordEnd + tail preference, capped before next speech; suggestedIn = firstWordStart − 0.3s lead margin, floored on prior audio. Every range is cross-checked against an isolated re-transcription (driftCheck) — disagreement raises INDEX_DRIFT and suppresses the suggestion. --manifest batch-verifies every scene of an edit.json, the expensive pass lint skips.

Reads → writesReads index + media → writes strips/cut-card sheets under work/candidates/; JSON verdict to stdout.

frame-sheettiled frames so the agent can see the footage

What it doesThe discovery eye: a tiled contact sheet of frames across a range. --scenes samples where the picture changes — the cheap way to find takes, resets, and b-roll ("look at 40 tiles, pick the coffee pour").

How it worksffmpeg frame extraction on a time grid (or at scene-change timestamps from the index), tiled into one PNG with timestamps burned per tile, sized to stay cheap as image tokens.

Reads → writesReads media (+ index for --scenes) → writes the sheet PNG; JSON with the path and grid to stdout.

timeline-sheetthe editor’s timeline as one image — the flagship artifact

What it doesThumbnails, a motion strip, the audio waveform with silence shading, non-speech events, the word-aligned transcript, and edit markers on one shared time axis. The agent reasons in tokens, then confirms the situation visually without guessing how one view maps to the other. --at zooms a moment.

How it worksRenders bands from the cached index onto a canvas: frame thumbnails on the ruler, motion curve, waveform with measured-silence shading, event and word lanes, and cut markers (the orange OUT). All bands share one time axis, so vertical alignment is meaning.

Reads → writesReads index + media → writes the sheet PNG; JSON with band legend to stdout.

lintpre-render cut-safety check for the whole manifest

What it doesThe cheap gate before rendering: every scene’s endpoints are checked from cached perception. Exit 1 when a blocking finding remains.

How it worksWalks edit.json scenes, resolves each source’s cached index, runs the same endpoint checks as candidates, and reports missing or degraded perception evidence. It never analyzes or writes.

Reads → writesReads edit.json + cached indexes → JSON verdict with per-scene findings; exit code is the contract.

cutrender the manifest — clips, cards, J/L cuts, music, assembly

What it doesThe hands. Renders each scene clip and the full assembly from edit.json: title cards, J/L-cuts bridged through card audio, dissolve/fadeblack transitions, a music bed with sidechain ducking and loudness normalization, per-scene gain, and 30ms de-pop micro-fades at every cut boundary by default.

How it worksBuilds ffmpeg filter graphs per scene and for the assembly: xfade/acrossfade for transitions (with pixel-format guards), aevalsrc/anullsrc card audio, sidechaincompress for ducking, afade micro-fades (J/L-aware, clamped for tiny clips), HDR-preserving encode with explicit color params, then a history snapshot of the manifest.

Reads → writesReads edit.json + media → writes clips/ and outputs/final_draft.mp4 + .ripple/history snapshot; JSON (paths, warnings, next: qa) to stdout.

qadeterministic delivery gates on the rendered file

What it doesThe involuntary playback review: decode integrity, color policy (HDR in = HDR out), clip counts, scene tails, dialogue loudness spread, leading/tail silence, black and freeze frames outside intentional regions, prompt-leak patterns, and expected scene endings verified against a transcript of the render. --report renders the HTML QA report.

How it worksffprobe/ffmpeg probes (blackdetect, freezedetect, volumedetect, loudnorm), manifest-aware intentional-region exclusion (cards, transitions), transcript-backed content gates that refuse to pass unverified (fail-unverified plan), whitespace-normalized ending matching, trend snapshots across runs.

Reads → writesReads the render + edit.json (+ --transcribe) → JSON gate results; exit 1 on any failed gate.

Scale & multicam

hours of footage, repeated takes, synced cameras · 5 commands
searchfind where anyone says a phrase, word-accurate, across sources

What it doesPhrase search over every indexed source at word accuracy — the entry point for interview-at-scale work: find the answer inside hours, then verify it with candidates.

How it worksScans the cached word timelines (no re-transcription), matches normalized phrases, returns source, start/end, and surrounding words for each hit.

Reads → writesReads cached indexes → JSON hits with word-accurate times.

selectgroup repeated takes and recommend the best

What it doesTake selection on evidence: groups takes of the same line across files by transcript similarity and recommends the best per group — matched to script lines in fiction work.

How it worksClusters ranges whose transcripts overlap, scores candidates on measured signals (completeness, pace, fillers, energy), and reports the evidence behind each recommendation rather than a bare verdict.

Reads → writesReads cached indexes → JSON groups with per-take evidence.

syncmulticam offsets by audio cross-correlation

What it doesFinds how far each recording is offset from a reference so a moment found in one angle maps onto another. The measured answer to "which camera started when."

How it worksCross-correlates audio energy between files; reports offset (other_time + offset = ref_time), a normalized confidence (below ~0.2 the files likely don’t share audio), and a plain-language note about which file is missing head.

Reads → writesReads two or more media files → JSON offsets + confidence.

beatsa beat grid for the music bed

What it doesBPM, beat times, and confidence for a track — the timing lattice for cutting to music.

How it worksEnergy-flux onset detection in ffmpeg terms (zero-dependency floor); the prior-art ladder escalates to the aubiotrack binary when installed.

Reads → writesReads the audio/music file → JSON bpm + beat times.

studyextract taste from a reference edit

What it doesMeasures a reference edit’s cutting rhythm, delivery pace, tail preference, silence usage, energy, and grade lean, then proposes matching VIDEO.md values with the measurement behind each — taste captured as numbers, never applied without approval.

How it worksRuns the perception stack on the reference, derives distributions (shot lengths, tail lengths, wps), maps them onto VIDEO.md fields, and shows the evidence per proposal.

Reads → writesReads a reference video → JSON proposed VIDEO.md values + evidence; writes nothing without approval.

Support

inspection, transcripts, history, hand-off · 6 commands
doctorcheck the toolchain and print fixes

What it doesVerifies ffmpeg/ffprobe, whisper-cpp and its model, encoders, and optional binaries, and prints the exact install commands for anything missing.

How it worksProbes PATH and versions, runs capability checks (e.g. whisper --split-on-word support, zscale availability), and emits guided fixes.

Reads → writesReads the environment → JSON status + fix hints.

probeinspect a file’s streams — or the whole media bin

What it doesPer-file: streams, resolution, fps, HDR/color metadata, audio layout, capability warnings. With no file: the media bin listing plus each source’s perception-index state — the orientation command for a cold session.

How it worksffprobe JSON parsed into a normalized envelope; HDR detection from color_transfer/primaries; index state from the analysis cache.

Reads → writesReads media/cache → JSON description.

historysave, list, and diff cut snapshots

What it doesThe edit’s memory: every cut auto-snapshots the manifest; history lists versions, diffs two of them scene-by-scene, and dedups identical states.

How it worksContent-hashes the manifest; stores under .ripple/history; --diff compares manifests structurally (scenes added/removed/retimed) rather than as text.

Reads → writesReads/writes .ripple/history → JSON list/diff.

captionsword-accurate captions in output time

What it doesSRT plus styled ASS captions whose times are in the rendered output’s timeline (not source time), with optional burn-in.

How it worksMaps source-time word timings through the manifest’s scene arithmetic into output time, then emits caption files; burn-in re-renders with the subtitles filter.

Reads → writesReads edit.json + index → writes .srt/.ass (and optionally a burned render).

handoffhand the cut to an NLE

What it doesExports the cut as OTIO (Resolve-native since 18.5), FCP7 XML (the Premiere path — its OTIO import is still beta), FCPXML (Final Cut Pro), or EDL (universal). For interview-scale work the NLE is a peer terminal: cut the structure right, hand over the taste-heavy 20%.

How it worksTranslates edit.json scene arithmetic into each format’s timeline model directly — no OTIO library dependency; ripple’s render path is its own ffmpeg bridge.

Reads → writesReads edit.json → writes .otio/.xml/.edl.

transcribetranscript for any file — subtitles first, whisper fallback

What it doesThe plain transcript utility: existing subtitle tracks win (free and exact), whisper-cpp otherwise, cached; --words adds word-level timing.

How it worksSame chunked whisper pipeline as analyze (shared cache with version/mode/model metadata so stale entries can’t be reused); subtitle extraction via ffmpeg when a track exists.

Reads → writesReads media → writes .txt/.srt/word JSON under work/transcripts/; JSON with file paths.

video-use (browser-use)17k★ · MIT · TypeScriptADJACENT

The closest neighbor, and a well-designed one. Same thesis, stated their way: "the LLM never watches the video. It reads it." The bets differ: video-use packs one cloud transcription call (ElevenLabs) into ~12KB of markdown; Ripple builds a local-first perception index with whisper-cpp and ffmpeg on your machine. Ripple's always-on 30ms cut fades credit video-use's design. What Ripple adds above it: verified cut endpoints, deterministic QA gates, and NLE handoff — the "check your work" half of editing.

OpenMontage~39.5k★ · AGPLDIFFERENT PRODUCT

Generation-first: a provider menu (Runway, Kling, Veo…) that composes generated shots into montages. Ripple is the opposite bet — editing-first and opinionated: it cuts footage you already have and routes generation out to a deliberately chosen stack. Its named-failure-mode QA scoring is a kindred idea to Ripple's deterministic checks. AGPL vs Ripple's Apache-2.0.

auto-editor4.6k★ · very active · NimTECHNIQUE CREDIT

The best deterministic batch silence-cutter: one command, one job, done instantly — not agentic, and it makes no editorial decisions. Ripple's cut-safety guards port its ideas with credit: asymmetric lead/trail margins (Ripple's suggestIn) and minimum cut/clip durations (Ripple's STUTTER_CUT/MICRO_CLIP flags). Use auto-editor when you want fast unattended silence removal; use Ripple when the cut needs judgment and verification.

WhisperX23.1k★ · active · Python/PyTorchTECHNIQUE CREDIT

The gold standard for long-source transcription alignment: VAD pre-chunking plus wav2vec2 forced alignment (word boundaries to tens of ms). Ripple ports the chunking idea into its zero-Python whisper-cpp pipeline — silence-anchored ≤30s windows, offset-added — so cumulative drift is structurally impossible without the ~2GB PyTorch install. WhisperX stops at transcription; Ripple is the editing loop above it.

videogrep · PySceneDetect · aubiothe technique lineageTECHNIQUE CREDIT

Three more ideas Ripple stands on: videogrep pioneered transcript-driven cutting (Ripple's search keeps its two-tier contract — sentence-safe by default, word-exact on request); PySceneDetect's adaptive rolling-ratio scene test informs how Ripple reads scene scores; aubio's aubiotrack is the planned optional beat-grid backend, the same detected-binary pattern as whisper-cpp.

FunClip · ClipsAI · ButterCutAlibaba · OSS · PolyForm-NCADJACENT

The adjacent category. FunClip: clip-by-selecting-transcript-text as the primitive. ClipsAI: speaker-tracking auto-reframe to 9:16 — finishing-suite work Ripple leaves to the NLE. ButterCut: the same "agent edits, NLE finishes" thesis as Ripple's handoff — independent confirmation that terminal is right.

HyperFrames · Remotion · ElevenLabs · Gemini Imagethe routed-to stackROUTES TO

Not competitors — the production stack Ripple's playbooks choose deliberately: HyperFrames for motion graphics from scratch, Remotion for timed React overlays (timed from Ripple's word-level transcript), ElevenLabs for voice-over and music beds generated to the manifest's exact duration, Gemini for stills and cards. Ripple owns assembly timing and verification; they own creation.

The strategy difference in one line: ffmpeg is hands, HyperFrames is imagination, the NLE is the finishing suite — Ripple is the eyes, ears, and editorial judgment loop. Everyone above owns one slice; Ripple owns the loop that connects them and routes to the rest.

Measured on ripple’s own eval harness (reproducible via evals/)
  • Same agent, same task, tooling ablated: Sonnet with raw ffmpeg: 559s / $1.12. With the ripple CLI alone: 158–249s / $0.76–1.11 across two release runs — 2–3.5× faster, and it passes the deterministic checks (endings kept, tails tight, HDR preserved) that the bare run barely scrapes.
  • All four scenarios pass live evals (v0.11): flagship tighten, demo UI-settle from motion curves, interview search in 7.4 minutes, fiction J-cut, multicam sync (24s / $0.21), animation routing.
  • The opinions route correctly (v0.12): service evals pass 10/10 checks — VO → ElevenLabs with a standing voice ID, music → generated to the manifest's duration, b-roll → recut/stock before generation.
  • Triggering: 19/20 against ~20 competing installed video skills — plain-language editing asks route into ripple 3/3 without ever naming it; near-misses stay out.
  • Channel economics: for text-domain questions the index is 4× cheaper than the sheet; behavior questions (the look-down) are decided in the image channel — each question has a home channel.
Honesty box · what is NOT measured
  • No head-to-head benchmark against these tools exists — ours or anyone's. Published video-editing benchmarks target generative editing; nobody scores editorial quality (open ground the prior-art doc flags as ours to take).
  • The fairest proxy we have is the ablation above: rung A ≈ "an agent with no editing tool," which is the world most of these tools also try to improve on — but that is a proxy, not a comparison run against video-use or OpenMontage themselves.
  • Borrowable deterministic metrics for a future benchmark: cut-point-vs-word-boundary distance, boundary embedding distance, audio RMS at cuts, SyncNet offline.
