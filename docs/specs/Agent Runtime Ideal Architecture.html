Architecture contract

Agent runs should feel like a calm project activity feed, not a coding terminal.

Ripple can lean on AI SDK for chat mechanics while preserving its own product meaning. The durable event log stays technical and replayable. The shared runtime model turns that work into language a motion designer understands. Chat and comments render the same truth with different density.

AI SDK for chat transport agent_run_events as truth Ripple owns product language chat and comments stay in sync
01

Default user promise

Show what is happening in the project: thinking, looking at frames, exploring the project, editing the composition, checking the work, and preparing changes.

Hidden by default

Collapsed rows hide provider names, raw tool names, absolute paths, stdout, stderr, MCP servers, and JSON payloads. Opened umbrellas can show sanitized event receipts like files read, files edited, and commands run.

Always preserved

Raw provider events, provider refs, sequence order, approvals, file changes, and replay metadata remain available for diagnostics and support.

05

Event-to-UI mapping

The mapping keeps raw provider truth intact while giving collapsed rows a small verb vocabulary. Open umbrellas should show the evidence that caused the verb.

Tense is part of the contract: the currently active item uses an active verb like Editing, while completed evidence inside the same umbrella switches to past tense like Looked, Explored, and Edited. When the umbrella is closed, only the umbrella title is visible and shimmering. If the user opens it, the active detail row shimmers too.

Chat and comments use the same product row. Chat can show the full ordered trail; comments usually show only the latest collapsed row plus the proposed-change state. Comment status lines use the Active row column, not the opened-umbrella detail text. When the comment is ready, the comment reply is the agent's final message to the user.

Chat is allowed to interleave compact umbrellas with assistant narration. If the agent looks, explains what it saw, edits, explains again, then verifies, those should stay in transcript order as several closed umbrellas between messages rather than being forced into one global turn summary.

Grouping rule: a collapsed umbrella shows one dominant activity, such as Editing. Supporting work like looked frames, searched files, and read files appears as muted meta or opened detail rows. Completed umbrellas use a dominant outcome, such as Edited, with supporting counts like "looked at 2 frames" kept secondary.

08

Current vs ideal coverage

Likely already built

  • AI SDK chat state and transport path.
  • Runtime event persistence and replay source.
  • Chat activity feed component family.
  • Comment thread and composer surface.

Needs tightening

  • One shared semantic event and projector contract for chat and comments.
  • Consistent tense: active "-ing", completed past tense.
  • Closed umbrellas as the default state.
  • Raw provider wording kept out of projected UI summaries.
  • Approval, cancellation, interruption, and comment edge states.

Needs eval proof

  • Active row and active detail shimmer while thinking or editing.
  • Comment final line is the agent reply, not generic "changes ready" copy.
  • Reload produces the same rows without duplicating messages or side effects.
  • Unknown tools use neutral project activity, not false edit/check labels.
  • Provider-specific Codex and Claude fixtures normalize to the same Ripple vocabulary.
09

Eval matrix

Each row in this document should either be covered by a fixture, a component test, or a live e2e run. Missing tests become build tasks.

Projection fixtures

  • Provider events collapse into semantic product rows.
  • Raw tool names stay in evidence, not titles.
  • Provider refs survive replay.
  • Absolute paths render as basenames or project-relative paths only.
  • Unknown events fall back to project_activity.

Chat UI fixtures

  • Collapsed umbrellas are default.
  • Pending row shimmers; completed row does not.
  • Final assistant reply remains visible.
  • Opened editing rows show file names and tiny diff counts.
  • Approval waiting state is visibly blocked, not shimmering work.

Comment UI fixtures

  • Pending comment uses latest shared activity row.
  • Ready comment shows the final agent reply.
  • Comment and chat agree on status.
  • Comment status is scoped to the same run and comment id.
  • Error, approval, cancelled, interrupted, retry, and resume states are explicit.

Replay and errors

  • Reload reconstructs rows without duplicated text.
  • Errors are visible and do not look like progress.
  • Approval prompts include enough scope to approve confidently.
  • Reconnect streams only unseen event ids.
  • Replay never re-runs tools or provider side effects.

Migration plan

1. Lock fixtures

Create failing projection, chat, comment, approval, replay, and provider fixtures from this document before changing runtime behavior.

2. Consolidate projector

Move semantic facts, grouping, tense, detail text, and support references into the shared runtime summary/projector layer.

3. Build against proof

Update chat, comments, approvals, replay, and provider adapters only after the evals describe the intended outcome.
