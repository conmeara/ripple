Ripple Spec / Visual Context Eval / May 2026

Fast visual context for agents

This page is trimmed to the work we actually tested: capture latency, fidelity, normalization, and the resulting product split between current-frame capture and render-backed arbitrary frames.

Decision

So what
Use the live preview for the current frame. Use warm Engine renders for everything else.

Current-frame capture is fast enough to make visual context feel immediate. Arbitrary timestamps and frame sheets need source-canonical rendering, not display pixels.

Routing rule
  • CurrentcapturePage() on the live preview iframe, with normalization guards.
  • CertainWarm HyperFrames Engine render at the requested timestamp.
  • SheetWarm Engine sampled renders assembled into a compact frame sheet.

The Three Needs

Need 01
Current frame

What the user is looking at right now. The pixels already exist in the app preview.

Use live preview capture. capturePage() is fast and faithful to the user's visible surface. Normalize dimensions and mark it as display context.
Need 02
Certain frame

A requested timestamp in a known composition. The user may not be looking at that timestamp now.

Use warm Engine render. Do not seek the user's visible preview. Render source-canonical pixels from project, composition, revision, and time.
Need 03
Frame sheet

A compact grid across a time range so the agent can reason about motion, timing, and transitions.

Use warm Engine sampled renders. Capture a small set of timestamps, assemble a sheet, and include a manifest mapping cells to times.

Implementation Read

Case Agent-facing command Backend route Reason
Current app view ripple snapshot --at current --json Live preview capturePage() Fastest path; reflects what the user sees right now.
Specific timestamp ripple snapshot --at 1.25s --json Warm Engine render Source-canonical and not dependent on the visible preview state.
Motion over time ripple frame-sheet --range 0s..8s --samples 8 --columns 4 --json Warm Engine renders plus sheet assembly Best tradeoff for agent reasoning about timing without large payloads.
Guardrail

Do not ask agents to choose browser plumbing. Agents should use the app-managed ripple visual commands. Ripple should route those commands to the fastest correct backend and return local image paths plus manifest JSON.
