Snapshot or frame sheet captured. Currently shows as a generic MCP tool call with a thumbnail.

event: status → data-agent-runtime (kind: status)
Tiny muted line — filtered to only pass when it has a meaningful label. Most status events are dropped silently.

File edit / write
event: tool_start/end → tool-Edit / tool-Write
Shows file path and a unified diff. Plan files (.plan.md) get a different card layout.
Bash command
event: tool_start/end → tool-Bash
Command run, exit code, stdout/stderr. ripple snapshot and frame-sheet are auto-approved (no interruption).

When a final text reply exists, the tool steps before it collapse into "N steps." Tap to expand.
Agent reply (text)
event: assistant_text_delta → text part → MemoizedTextPart
The markdown response. Includes code blocks, lists, inline code. Always shown when present.
Token usage
event: usage → message-metadata → AgentMessageUsage
Input/output tokens, duration, cost. Currently shown as a faint line below each response.
Interruptions
Approval request
event: approval_request → data-agent-runtime → AgentRuntimeApprovalCard
Agent pauses and asks permission. Approve/deny resumes or stops the run. ripple visual tools are auto-approved and never trigger this.
Not shown above: task sub-agents, web search, web fetch, ask-user-question, plan cards, Mermaid diagrams — present in the codebase but don't appear in typical Ripple composition runs.
