01
System Prompt

What the AI is told about its world

The system prompt is assembled dynamically each run from three stacked layers. For Claude it becomes the systemPrompt field in sdk.query(); for Codex it becomes developerInstructions.

appPolicy string
Layer 1 · Prepended first
Visual Tool-Choice Policy
Tells the AI exactly when and how to use Ripple's native visual tools — before reaching for shell commands, browser tools, or generic screenshot utilities.
"When a user or comment asks for visual context, make the native Ripple visual tool the first external action."
· Use snapshot at current for the visible frame
· Use snapshot at 1.25s only for an exact-time request
· Use frame sheet for motion over time or a time range
· Comment runs may already have auto-attached visuals — use those first, only call native tool for a fresh visual
· Normal chats do not get auto images — request on demand
Layer 2 · Core identity
RIPPLE_PROVIDER_POLICY
The main behavioral policy — defines who the AI is, what it's working on, and what rules it must follow. Sourced from ripple-provider-policy.ts.
"You are operating inside Ripple, a local-first motion graphics editor for HyperFrames projects."

· Treat compositions as plain HTML/CSS/JavaScript with GSAP timelines on window.__timelines
· Respect workspace boundaries — never edit Main from a revision workspace
· Prefer bundled HyperFrames skills; do not use web search for ordinary local edits
· Treat runtime context as source of truth for active composition, frame/time, preview source
· Do not write transient state into AGENTS.md or CLAUDE.md
· Speak like a motion-editor assistant — hide Git/branches/terminals unless asked
Layer 3 · Capabilities map
Skills Info Block
Lists available HyperFrames skills and their filesystem paths, plus guidance on the ripple-visual-context skill. Tells the AI to prefer these over npx/bunx/package installs.
hyperframes
hyperframes-cli
gsap
ripple-visual-context
"Use app-managed bare commands (ripple, hyperframes) instead of npx, bunx, or package installs."
"Use it proactively after creating or editing visible motion work."

Runtime Context

Per-run context injection

Beyond the static system prompt, each run injects dynamic context that tells the AI exactly where it is and what it's working on. This is the AI's "situational awareness" — assembled fresh for every conversation turn.

projectId
Identifies which Ripple project the AI is operating in.
compositionId
The active composition being edited or reviewed.
previewTimeSeconds
Current playhead position in the preview.
previewFrame
Current frame number at the playhead.
commentThreadId
Set when the run is triggered from a comment thread.
revisionId
Set when the run targets an isolated revision workspace.
CLAUDE.md / AGENTS.md
Project notes injected as context. Falls back to main project copy when running in a revision workspace.
Attachments
User-dropped images (PNG/JPEG/WebP/GIF) and documents (PDF, plain text) — base64-encoded into message content blocks.
@mentions
User can mention @[file:path], @[skill:name], @[tool:name], @[agent:name], @[folder:path] — contents injected into prompt.
previewSource — determines which workspace the AI targets
main
Normal chat — edits go into the primary project workspace
comment-revision
Comment-triggered run — isolated revision workspace, cannot touch Main
chat-worktree
Chat-based edit in a further-isolated git worktree
export
Run scoped to an export job target
Frame comment visual context
A still frame is automatically attached at run-start — AI should use it first before calling ripple_snapshot
Range comment visual context
A frame sheet over the comment's time range is automatically attached — AI should use it first before calling ripple_frame_sheet
03
Tools

What the AI can call

The AI has access to three layers of tools: Ripple's two native visual tools (the primary way the AI sees compositions), the full claude_code preset toolset, and MCP servers loaded from the user's config and installed plugins.

ripple_snapshot
Single Frame
Captures one frame from the active composition as an image. Returns pixels directly in the tool result — the AI sees the actual rendered frame, not just source code. Auto-approved; never needs user permission.
param type description
at string "current" for visible app frame · "1.25s" for exact timestamp · default: "current"
composition string? Project-relative HTML path. Omit for active/default composition.
Returns
Image content block (base64 PNG/WebP) + JSON metadata with path, mediaType, bytes. Image appears directly in tool result — AI sees the pixels.
auto-approved Bash(ripple snapshot) · Bash(ripple snapshot *)
ripple_frame_sheet
Motion Over Time
Captures a grid of evenly-spaced frames across a time range — like a film contact sheet. Lets the AI understand motion, transitions, and timing across the composition without scrubbing frame by frame. Auto-approved.
param type description
range string Time range like "0s..8s". Use user's range, comment range, or compact overview. Default: "0s..8s"
samples integer Number of evenly-spaced frames. 3 for short eval ranges, 8 for normal motion overview. Default: 8
columns integer Grid columns. 3 for three samples, 4 for eight-sample overview. Default: 4
composition string? Project-relative HTML path. Omit for active/default composition.
Returns
Sheet image (base64 PNG/WebP) + JSON metadata. AI sees all sampled frames in a single image — ideal for understanding animation arcs, easing, and transitions.
auto-approved Bash(ripple frame-sheet) · Bash(ripple frame-sheet *)
claude_code preset
Full Claude Code toolset: file read/write/edit, bash, web search, etc. For Claude only — tools: {type:"preset", preset:"claude_code"}
MCP Servers
Merged from: ~/.claude/claude.json · project .claude/claude.json · mcp.json · installed plugins. OAuth tokens refreshed per-run.
HyperFrames Skills
App-managed skills loaded at: resources/claude-plugins/ripple-visual-context/skills/ and resources/agent-skills/

05
Workspace

Isolation boundaries

Each run is scoped to a specific workspace. The AI is told which workspace it's in via previewSource and is strictly prevented from crossing boundaries — especially from ever editing Main while in a revision workspace.

Least isolated
Main Workspace
Normal chat edits. Direct modifications to the primary project files. No isolation boundary — changes are immediate.
previewSource: main
→
Revision isolation
Revision Workspace
Triggered from a comment thread. Isolated copy — the AI cannot edit Main. Changes go into a revision branch for review.
previewSource: comment-revision
→
Most isolated
Chat Worktree
Further isolated git worktree for chat-based edits. Allows exploratory changes without affecting the main or revision branch.
previewSource: chat-worktree
AGENTS.md fallback: When running in a revision workspace, the AI's project notes (CLAUDE.md / AGENTS.md) are read from the revision workspace first. If not found there, the main project's copy is injected as baseInstructions (Codex) or a fallback instructions block (Claude) — ensuring the AI always has project context even in isolated workspaces.
06
Events

What the AI emits back to the UI

Both providers stream structured events through AgentProviderEventSink. The UI consumes these to update the chat, show tool activity, request user approvals, and render file changes in real time.

Event stream
tool_start
AI is about to call a tool — shows the tool name and input arguments in the UI
tool_update
Streaming progress from a tool call (e.g. partial output)
tool_end
Tool call completed — includes result/output shown to user
assistant_message
Text response from the AI streaming to the chat panel
reasoning
Extended thinking content (when enabled) — may be collapsed in UI
file_change
AI edited a file — triggers diff view and preview refresh
status
Run lifecycle updates: starting, running, complete, error
usage
Token/cost usage for the run — shown in session info
approval_request
AI wants to use a tool that needs user confirmation — UI shows approve/deny prompt. Visual tools are auto-approved and never trigger this.
Activity kinds — shown in the UI as status indicators

Approval rules
Auto-approved: ripple snapshot and ripple frame-sheet commands never interrupt the user.
Needs approval: All other tool use routes through canUseTool (Claude) or requestApproval (Codex) — the UI shows an approve/deny prompt and the run pauses until the user responds.
